Introduction

This project demonstrates the usage of Concordion extensions with Selenium2 (also known as WebDriver).

  1. LoggingDemo demonstrates the LoggingTooltipExtension. The log output is written by the SeleniumEventLogger class, which acts as a WebDriverEventListener.
  2. ScreenshotDemo demonstrates the ScreenshotExtension. The SeleniumScreenshotTaker class uses Selenium's TakesScreenshot interface to take a custom screenshot of the web page.

Running the Concordion tests

The tests use Selenium's FirefoxDriver, so rely on Firefox being installed.

You can run build the tests using either Gradle or Maven.

Using Gradle

  1. Download and install Gradle 0.9
  2. Unzip this package
  3. From a command line:
    • gradle test
  4. view the Concordion output under the subfolder 'build/reports/spec/org/concordion/ext/demo/selenium/' of the project folder

Using Maven

  1. Download and install maven
  2. Unzip this package
  3. From a command line:
    • mvn test
  4. view the Concordion output under the subfolder 'target/concordion/org/concordion/ext/demo/selenium/' of the project folder

Generating an Eclipse Project

If you want to generate an Eclipse project:

  1. Download and install gradle 0.9
  2. Unzip this package
  3. From a command line:
    • gradle eclipse
  4. import this project into Eclipse (File > Import > Existing Project into Workspace, select this project)
  5. run the tests LoggingDemoTest and ScreenshotDemoTest using Run As > JUnit Test

The dependencies are listed in build.gradle, and will be downloaded by Gradle.
